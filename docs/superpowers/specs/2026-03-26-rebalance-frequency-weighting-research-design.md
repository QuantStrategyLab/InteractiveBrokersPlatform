# Rebalance Frequency and Weighting Research Design

## 1. 背景

当前默认研究基线已经切到统一总池 `VOO + XLK + SMH` 版本，规则是：

1. 季度调仓
2. `Top 2`
3. 等权持有
4. 一次性切到目标仓位

这一版长期表现已经明显优于之前的 `QQQ` 默认版，但还有两个自然问题没有回答：

1. 调仓频率改成更快或更慢，会不会更好
2. `Top N` 确认后，是否应该继续等权，还是按动量强弱分配仓位

本轮先只研究这两件事，不碰分步加仓/减仓。

## 2. 目标

这轮研究要回答：

1. `月度 / 季度 / 半年` 调仓，哪种频率更适合当前统一总池
2. `Top 1 / Top 2 / Top 3` 是否优于当前 `Top 2`
3. `Top 2` 下，等权和按动量分配谁更合理
4. 如果收益改善，是否值得继续进入下一轮研究

## 3. 非目标

这轮先不做：

1. 不改实盘默认策略
2. 不改 ETF 池
3. 不改动量公式、SMA200、金丝雀规则
4. 不做分步加仓/减仓
5. 不引入交易成本
6. 不做更细参数网格

## 4. 研究方案

### 4.1 基线

以当前默认研究基线作为对照：

- `quarterly_top2_equal`

即：

1. 季度调仓
2. `Top 2`
3. 等权配置

### 4.2 实验矩阵

本轮只测 6 组：

1. `monthly_top2_equal`
2. `semiannual_top2_equal`
3. `quarterly_top1_equal`
4. `quarterly_top3_equal`
5. `quarterly_top2_momentum_weighted`
6. `monthly_top2_momentum_weighted`

控制原则：

1. 频率只测 `月 / 季 / 半年`
2. `Top N` 只测 `1 / 2 / 3`
3. 权重只测 `等权` 和 `按动量比分配`

## 5. 回测规则

沿用现有研究脚本规则：

1. 13612W 动量
2. `SMA200` 趋势过滤
3. 持仓 `+2%` bonus
4. 金丝雀应急
5. 空位补 `BIL`

仅允许修改：

1. 调仓月份集合
2. `Top N`
3. 权重分配模式

## 6. 输出结果

### 6.1 绩效指标

继续输出：

1. Total Return
2. CAGR
3. Max Drawdown
4. Volatility
5. Sharpe
6. 与 `SPY` / `QQQ` 的相关性

### 6.2 关键阶段

继续看：

1. 全样本公共区间
2. `2022`
3. `2023+`

### 6.3 执行指标

新增两项：

1. 调仓次数
2. 近似换手率

这两项用来判断：

1. 更高收益是不是只是靠更频繁交易换来的
2. 更低频率是不是只是少交易但没有真正改善风险收益

## 7. 技术实现

优先继续复用：

- `research/backtest_qqq_variants.py`

不新开脚本。

新增配置参数：

1. `rebalance_months_override`
2. `top_n_override`
3. `weighting_mode`

其中 `weighting_mode` 先只支持：

1. `equal`
2. `momentum`

## 8. 风险

1. 月度调仓可能只会显著提高换手，不一定提高净收益
2. `Top 1` 可能提高收益上限，但波动和路径依赖可能更强
3. `Top 3` 可能更稳，但也可能稀释强势资产
4. 动量加权如果表现更好，可能只是因为少数年份强势标的集中带来的样本优势

## 9. 验收标准

如果满足下面任一条件，才值得继续研究：

1. 全样本优于基线，且 `2022` 不明显变差
2. `2023+` 表现更好，同时换手没有明显失控
3. 风险收益比有改善，不只是单纯提高收益

如果所有实验都没有稳定优于基线，就停止继续细调。
